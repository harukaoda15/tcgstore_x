# tcgstore-x 仕様書

最終更新: 2026-03-14

## 更新ルール

- 本プロジェクトの仕様調整を行ったら、同じ変更をこのファイルにも反映する。
- 投稿文ルール、判定条件、Cron、手動テスト方法を変更した場合は必ず更新する。

## システム概要

- 実行基盤: Cloudflare Workers
- スケジュール: Cloudflare Cron Triggers
- 永続化: Cloudflare KV（`STATE`）
- 投稿先: X (Twitter)
- AI生成: Anthropic Claude API

## 現在の運用戦略（要点）

- 目的は「広告感の強い定型投稿」ではなく、読み手の判断材料になる投稿を継続すること。
- dailyは `TCG STORE` / `メルカリくじ` を交互寄りに出し、同一URLの再投稿を避ける。
- `price_spike` で取得した直近相場カード（72時間以内）を daily 候補優先に利用し、相場関心と投稿対象を連動させる。
- 固定テンプレ投稿は停止。AI生成に失敗した場合は無理に投稿せずスキップし、ランダムdailyは他ソースへフォールバックする。
- 文面は「ユーザー利益・事実先頭」「絵文字最小（0〜1）」「煽り抑制」を基本方針とする。

## 実行スケジュール

- `*/30 * * * *`
  - 5%/1% しきい値監視（メルカリくじ + TCG STORE）
- `0 3 * * *`（JST 12:00）
  - daily 紹介投稿（`TCG STORE` / `メルカリくじ` をスロットseedで選択 + 連投偏り防止）
- `0 12 * * *`（JST 21:00）
  - daily 紹介投稿（`TCG STORE` / `メルカリくじ` をスロットseedで選択 + 連投偏り防止）

## 機能1: 5% / 1% しきい値監視

- 判定対象: 取得データの残数/総数から算出した割合
- 対象ソース: `メルカリくじ` と `TCG STORE` の両方
- 5%以下: `🎯` テンプレート
- 1%以下: `🚨` テンプレート
- しきい値投稿済みフラグを KV で管理し、重複投稿を防止
- ラスト賞画像がない商品は、ラスト賞訴求行を自動で省略
- メルカリくじの文体は「孤高のラッパー」トーン（静か・冷たい・少し挑発的）
  - 短文中心、過剰説明しない
  - `😮‍💨 😏 👀 🫥 👁️ 😶‍🌫️ 🫠 🫰` を少数使用
  - 安直な煽り語（例: 焦れ/急げ/爆アツ）は使わない

### 手動確認

- プレビュー: `/?forceLevel=under_5` または `/?forceLevel=under_1`
- 実投稿: `/?commit=1&forceLevel=under_5` など

## 機能2: TCG STORE daily 紹介（AI）

### 候補抽出条件

- 必須:
  - `id`, `name`, `price`, `url`, `mainImageUrl`
  - `visibility=true`
  - `status=1`
  - `stock > 0`
- 除外:
  - 売切れ、停止、主要項目欠損

### 選定ロジック

- 基本: `JST日付seed + JST時間帯seed` を使って候補を選定
- 直近投稿URLと同じ場合は回避
- 直近投稿履歴（最大10件）に含まれるURLも可能な限り回避
- 直近の `price_spike` 情報（72時間以内）がある場合、該当カード名に関連する商品を優先
- `pick_offset` 指定時は候補オフセットを追加
- `exclude_unit` 指定時は該当通貨を候補除外

### 通貨判定

- `currency` と商品名から判定:
  - `pOAS`
  - `OAS`
  - `コイン`

### 画像ルール

- daily 投稿はメイン画像1枚のみ添付
- 画像未取得の商品は候補から除外
- メルカリくじは汎用OG画像（`openGraph.png` 等）を避け、商品キービジュアルを優先取得する

## 機能3: メルカリくじ daily 紹介

- 候補: メルカリくじ一覧から取得した販売中URL
- 選定: JST日付seed + JST時間帯seed + 直近履歴回避（TCG STOREと同様）
- 直近の `price_spike` 情報（72時間以内）がある場合、関連キーワードを含むくじを優先
- 投稿形式: **AI生成（固定テンプレ廃止）**
  - 2〜4行、最終行は `{商品URL}` のみ
  - 1行目は商品名始まり禁止（読み手メリット/事実始まり）
  - ハッシュタグ禁止、絵文字は0〜1個まで
- 画像: メイン画像1枚
- トーン:
  - 5%/1%追い込み投稿とは分離
  - 通常紹介は事実ベースで簡潔、露骨な煽りはしない

## AI生成仕様

### 使用データ

- TCG STORE API:
  - 商品名、価格、在庫、総口数、残り率、1日上限
  - 上位賞/次点賞のカード名
  - coin表記賞品の最小〜最大値（抽出できる場合）
  - **各ランク当選確率** (`card.supply_amount / oripa.supply`)
    - ランクごとの総枚数・確率%・代表カード名を算出
    - 1等+2等合計確率も算出

### パターン

- `market_analysis`
- `contrarian`
- `comparison`
- `trivia`
- `urgency`

### パターン制御

- `forcePattern` 指定時はそのパターン固定
- `urgency` は低在庫率時のみ有効（通常は強制回避）
- AI生成に失敗した場合は固定テンプレで投稿せず、`*_ai_failed` としてスキップ（ランダムdailyは他ソースへフォールバック）
- パターンはABテスト目的で残している（`market_analysis` / `contrarian` / `comparison` / `trivia` / `urgency`）。
  - 目的: 反応差を比較し、今後の最適化に使うため
  - ただし「テンプレ化」は狙っていない。テンプレ調を検出した場合はプロンプト/検証ルールを更新する

### 投稿文の冒頭ルール

- **1行目は商品名ではなく、読み手のメリット・数値的事実から始める**
  - 良い例: `最低保証1,100coin・赤字なしで引ける` / `1口999coinで〇〇狙い` / `1日1回限定で試せる`
  - NG例: `⚡黒炎の舞` のような商品名だけの冒頭
- 1等+2等合計確率は内部判断に利用。本文では `N人に1人` と `0.x%` の確率訴求を使わない
- カード名は投稿時に整形し、`[型番]` や `（拡張パック名）` などの注釈は除去する
- 1等+2等の確率は、条件を満たす場合 `1等と2等は約N人に1人` の言い回しを優先する（事実値ベース）

### 文字数モード（lengthMode）

- **short**: URL込み60〜90文字。1〜2文でサラッと要点だけ。
- **full**: URL込み100〜140文字。しっかり情報を盛り込む。
- `dateSeed % 2` で交互に切り替わり、タイムラインに緩急をつける。

### 文面ガード（検証）

- 文字数: lengthModeに応じて short=60〜90 / full=100〜140（X換算）
- URLは末尾
- ハッシュタグ規則厳守
- 絵文字は0〜1個まで（連打禁止）
- **冒頭が商品名のみで始まっている場合はリジェクト**
- 非pOAS商品で `pOAS/OAS/演出違い` 言及禁止
- 高在庫率で煽り文言禁止
- 根拠不明な「どんな層が引いてる」禁止
- 主観感想表現（例: `正直`, `個人的に`, `〜と思う`, `〜気がする`）禁止
- 1人1回限定商品のお得訴求を必須化
- **`N人に1人` 表現を禁止**
- **`0.x%` 表現を禁止**
- 確率を根拠にした煽り表現を禁止（例: 当たりやすい/狙いやすい/期待感が上がる）
- `1等+2等合計確率` は「上位賞の目安」として扱い、全体保証（`必ず` / `全員` / `100%`）に置き換えない
- **ランク名の解釈語を禁止**（例: `Legendary以上`, `Legendaryクラス`, `上位グレード`）
- **カード名の英単語を原文から補正しない**（API原文ベースで記載）
- 保証根拠がない `必ず/確定/100%` を禁止

## ハッシュタグルール（確定）

- `market_analysis` / `trivia` / `comparison` / `contrarian`:
  - 商品名から推定した関連タグを **最大1個** のみ付与
  - 例: `ブレヒロ` を含む商品は `#ブレヒロ`
  - 商品名で推定できない場合は、上位賞/次点賞カード名から推定
  - それでも推定できない場合は `#ポケカ` を1個付与
- `urgency`: `#ポケカ` のみ（1個固定）
- `pOAS / OAS`: なし

## 特記事項（強制訴求）

- `1人1回限定_1000coin`:
  - 1人1回限定
  - 最低1100coin以上訴求
  - 赤字覚悟/新規向け訴求を許容
- `1人1回限定_10000coin_ピカゼク`:
  - 1人1回限定
  - 最低10000coin以上訴求
  - 赤字覚悟/新規向け訴求を許容

## 手動テスト用エンドポイント

- daily プレビュー:
  - `/?mode=tcg_daily`
- daily 実投稿:
  - `/?mode=tcg_daily&commit=1`
- メルカリdaily プレビュー:
  - `/?mode=mercari_daily`
- メルカリdaily 実投稿:
  - `/?mode=mercari_daily&commit=1`
- 非pOASのみ:
  - `/?mode=tcg_daily&exclude_unit=pOAS,OAS`
- パターン固定:
  - `/?mode=tcg_daily&pattern=contrarian`
- 候補ずらし:
  - `/?mode=tcg_daily&pick_offset=3`
- サンプル一覧:
  - `/?mode=tcg_samples&count=5`
- price_spike プレビュー:
  - `/?mode=price_spike`（POST JSON / `commit` なし）
- price_spike 実投稿:
  - `/?mode=price_spike&commit=1`（POST JSON）

## 機能4: price_spike 投稿

- リクエストJSON:
  - `source: string`
  - `spikes: [{ card, variant?, card_id?, before, after, change_pct, fetched_at, period?, source_site?, source_url? }]`
- `spikes` の先頭1件のみ処理
- 入力バリデーション（厳格）:
  - 参照サイトは `snkrdunk` / `pokeca-chart` のみ許可
  - `source_url` は必須（対応サイトのURLのみ許可）
  - `variant` / `card_id` が不足している場合は `source_url` のページを取得してカード識別情報を補完
  - 補完は「カード名の近傍テキストで識別子が一意に取れた場合のみ」実施（曖昧なら投稿拒否）
  - カード識別子（`card_id`）または型番・収録番号等を含む `variant` が無い場合は投稿拒否
  - 曖昧名（例: 同名カード複数が存在しうる名称のみ）は投稿しない
- 重複抑止:
  - KVキー `price_spike:{source}:{card_id or canonical_card_name}`
  - 既に存在する場合は `skipped: true` で終了
  - 実投稿成功時のみTTL 21600秒（6時間）で記録
- 投稿文:
  - Anthropic 生成（失敗時はフォールバック文）
  - 人格: ポケカ好きな情報通が、相場の動きをさらっと共有するトーン
  - 構成:
    - 1行目: `カード名 + {period}で+{変化率}%`
      - `period` 未指定時は `直近` を使う（例: `直近で+8.08%`）
    - 2行目: `{前回価格}円 → {現在価格}円（+{変化率}%）`
    - 3行目: 背景の推察をやわらかく一文
    - URLは付与しない（市場情報のみ）
  - `#ポケカ` 固定
  - 60〜100文字
  - 煽り/主観/根拠なし予測を禁止
